# Governance

OSEF follows a Benevolent Dictator for Now (BDFN) model, transitioning towards a steering committee as the project scales.

## Maintainers
Currently, the core repository and architectural direction are managed by the founder. 
Maintainers are added based on sustained, high-quality contributions and alignment with the OSEF Constitution.

## Decision Making
- **Trivial changes** (typos, small bugs): Merged via standard PR review.
- **Major changes** (new subsystems, breaking changes): Require an RFC.

## The RFC Workflow
1. A community member opens an RFC Issue.
2. Discussion occurs over at least 1 week.
3. A maintainer approves or rejects the RFC.
4. If approved, an ADR is created, and development begins.

## Community Values
- **Technical Excellence**: We prioritize well-architected solutions over quick hacks.
- **Transparency**: All technical discussions must happen publicly in GitHub Issues or Discussions.
- **Respect**: We adhere strictly to our Code of Conduct.
