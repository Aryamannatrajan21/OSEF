# Engineering Philosophy

OSEF is built on a strict, immutable set of principles. We believe that an open-source project without a clear philosophy will eventually collapse under its own weight.

## 1. The Founder Manifesto
The initial vision for OSEF states that **Code is not text; it is a graph.** We believe that AI coding assistants are fundamentally flawed if they only operate on string matching or generic vector embeddings. True AI engineering requires an **Engineering Knowledge Graph**.

## 2. The Constitution
The Constitution defines the governance of the repository.
- No code is merged without a PR.
- No PR is merged without passing CI.
- No architectural change is made without an RFC.

## 3. Project Execution Directive (PED)
Our PED outlines our sprint cadence. We do not build features in a vacuum. Everything is scoped into discrete Sprints (e.g., Phase I: Platform Engineering, Phase II: Ecosystem Engineering).

## 4. Engineering Principles
- **Architecture First**: We write the RFC before we write the code.
- **Documentation as Code**: Documentation is not an afterthought; it is tested and versioned alongside the code.
- **Fail Fast, Fail Loud**: We prefer a hard crash over a silent failure.
