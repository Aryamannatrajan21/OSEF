# Glossary

Common acronyms and terminology used in the OSEF ecosystem.

- **ADR (Architecture Decision Record):** An immutable document capturing an architectural decision after an RFC is accepted.
- **Constitution:** The foundational governance rules of the repository.
- **EIL (Engineering Intelligence Layer):** The layer that interfaces the EKG with an AI agent (LLM), translating graph structures into contextual prompts.
- **EKG (Engineering Knowledge Graph):** A graph database representation of a codebase, connecting functions, classes, imports, and documentation.
- **Manifesto:** The founding document declaring that "Code is a graph."
- **OSTE (Open Source Transformation Engine):** The engine responsible for safely applying code modifications to the codebase.
- **Parser:** A subsystem that extracts an Abstract Syntax Tree (AST) from raw source files.
- **PED (Project Execution Directive):** The document outlining the sprint cadence and development strategy.
- **Plugin:** An independent package that extends OSEF's functionality (e.g., adding a new language parser).
- **Repository Intelligence:** The combined capability of the Scanner, Parser, EKG, and EIL.
- **RFC (Request for Comments):** A formal proposal for a major architectural change.
- **SDK (Software Development Kit):** Tools provided for writing custom Plugins.
- **Sprint:** A focused development phase (e.g., Phase I: Platform Engineering).
