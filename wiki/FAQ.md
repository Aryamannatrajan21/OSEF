# Frequently Asked Questions (FAQ)

### Why Python?
Python is the lingua franca of AI and data science. Since OSEF acts as the bridge between software repositories and Large Language Models, Python provides the best ecosystem for building the Engineering Intelligence Layer.

### Why an Engineering Knowledge Graph (EKG)?
Because string matching and simple RAG (Retrieval-Augmented Generation) are insufficient for complex software engineering. To safely modify an enterprise codebase, an AI needs to understand dependency graphs, typing constraints, and architectural contracts. The EKG provides this.

### How is OSEF different from traditional code generators?
Traditional tools focus on *generating new code* in isolation. OSEF focuses on *understanding existing code* so that AI can modify it safely without breaking the system.

### Why not modify repositories directly?
We believe in the Transformation Engine. AI agents should propose changes as a set of structured patches, which are then validated against the EKG before being written to disk.

### How are RFCs and ADRs used?
- **RFC (Request for Comments):** Used to propose a new idea and gather feedback.
- **ADR (Architecture Decision Record):** The final, immutable record of an architectural decision made after the RFC process concludes.
