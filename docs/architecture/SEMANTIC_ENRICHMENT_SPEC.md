# Semantic Enrichment Specification

The Semantic Enrichment Layer applies heuristics to the structural `SymbolTable` generated by language parsers.

## 1. Rules of Enrichment
- No modification of the source file.
- Heuristics must be additive (they attach data to `Symbol.metadata`).
- Enrichment logic belongs here, never in the Parser.

## 2. Examples of Inference
- A class containing `Service` in its name is flagged as `semantic_role = Service`.
- A method decorated with `@router.get` is flagged as `semantic_role = API_Endpoint`.
