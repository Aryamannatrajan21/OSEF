# OSEF Playbooks

## Mission
Playbooks bridge the gap between abstract architectural knowledge and concrete engineering tasks. They combine EKK rules, plugins, and DX principles into end-to-end, repeatable workflows.

If the EKK is the "dictionary," Playbooks are the "novels." They describe how to use OSEF to achieve specific, high-value outcomes.

## Available Playbooks

### Project Scaffolding
- [Python Library](python_library.md): Scaffolding and releasing a modern Python package.
- [Django SaaS](django_saas.md): Preparing a Django application for open-source (or secure internal) release.
- [FastAPI API](fastapi_api.md): Architecting a robust, scalable microservice.

### System Extension
- [Plugin Development](plugin_development.md): Creating, testing, and distributing an OSEF plugin.
- [GitHub Actions Integration](github_actions.md): Automating OSEF in your CI/CD pipeline.

### Transformation & Security
- [Repository Migration](repository_migration.md): Moving from legacy structures to OSEF standards.
- [Security Hardening](security_hardening.md): Auditing a codebase for vulnerabilities using OSEF.

### Community & Release
- [Release Process](release_process.md): Automating versioning, changelogs, and PyPI publishing.
- [Community Growth](community_growth.md): Building governance models to attract and retain contributors.
