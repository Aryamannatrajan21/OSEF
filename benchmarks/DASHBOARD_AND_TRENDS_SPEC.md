# Dashboard and Trends Specification

The OSEF Public Benchmark Dashboard provides the public evidence backing OSEF releases. 

## Data Source
The dashboard consumes the `dashboard.json` files generated by the `BenchmarkRunner` and archived in `benchmarks/history/`.

## Project Display Requirements
For each project in the Benchmark Corpus, the dashboard must display:
* **Repository & Language**
* **Scale:** File count, Graph Nodes, Graph Edges
* **Intelligence:** Correlations created, Policies evaluated, Violations found
* **Quality:** Engineering Confidence Score
* **Performance:** Execution Time, Memory Usage
* **Coverage:** Plugin Coverage, Domain Coverage
* **Status:** Certification Status (Pass/Fail)

## Historical Trend Charts
The dashboard must visualize performance and scale trends across OSEF releases.
Example trend comparison:
`v1.0.0` -> `v1.0.1` -> `v1.1.0` -> `v2.0.0`

Metrics to chart over time:
* Graph Generation Time (decreasing is better)
* Memory Consumption (decreasing is better)
* Node/Edge Density (increasing is better, assuming constant repository size)
* Engineering Confidence (increasing is better)

## CLI Integration
The dashboard data can be generated locally using the CLI:
```bash
osef benchmark dashboard
```
Initially, this command generates static HTML and JSON from local results. In the future, this data will feed a live web application.
